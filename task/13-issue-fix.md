# 前端页面测试问题修复 Issue / Codex Prompt

## 背景

当前正在测试 React 前端后台管理项目页面，发现多个交互、布局和展示问题。请在现有项目基础上进行修复和优化，保持现有技术栈、组件风格和后台管理系统的设计规范，不引入新的 UI 库。

请优先保证：

- 页面结构清晰
- 交互符合后台系统常见习惯
- 表格信息密度合理
- 不破坏现有路由、mock 数据和公共组件
- 修改后必须通过构建验证

## 一、菜单一级菜单折叠 / 展开优化

### 问题描述

当前侧边栏菜单中，一级菜单和二级菜单的层级展示不够完善。

例如：

- `系统管理` 是一级菜单
- 其下包含用户管理、角色管理、字典管理等二级菜单

目前一级菜单缺少明确的展开 / 收起能力。

### 修复要求

1. 一级菜单需要支持展开 / 收起。
2. 点击一级菜单时，可以展开或收起其下二级菜单。
3. 例如点击 `系统管理`，应能收起下面所有二级菜单。
4. 保持侧边栏整体视觉风格与现有项目一致。
5. 当前选中菜单状态不能丢失。
6. 如果当前路由属于某个一级菜单下的二级菜单，该一级菜单应默认展开。

### 验收标准

- 一级菜单可以正常展开 / 收起。
- 二级菜单显示和隐藏逻辑正确。
- 当前页面对应菜单仍然可以正确高亮。
- 页面刷新后，当前路由所属一级菜单默认展开。
- 执行 `npm run build` 通过。

## 二、右上角退出登录入口重复优化

### 问题描述

当前页面右上角存在两个退出登录入口：

1. 顶部右上角单独的 `退出登录` 按钮
2. 用户头像 / 用户信息下拉菜单中的 `退出登录`

两个入口重复，页面显得冗余。

### 修复要求

1. 移除顶部右上角单独展示的 `退出登录` 按钮。
2. 只保留用户头像 / 用户信息下拉菜单中的 `退出登录` 选项。
3. 用户点击头像或用户信息区域后，下拉菜单中应包含 `退出登录`。
4. 退出登录逻辑保持不变：
   - 清除登录状态
   - 返回登录页
   - 不影响路由守卫逻辑

### 验收标准

- 顶部右上角不再单独显示 `退出登录` 按钮。
- 用户头像 / 用户信息下拉菜单中仍可正常退出登录。
- 退出登录后能够正常跳转登录页。
- 执行 `npm run build` 通过。

## 三、字典管理页面布局与交互优化

### 问题描述

当前字典管理页面采用左侧字典类型、右侧字典项的布局，但初始页面直接显示字典项，视觉上不够清晰。

期望改为：

- 初始只显示完整的字典类型列表
- 选择某个字典类型后，再打开右侧字典项区域
- 右侧字典项区域可关闭
- 关闭后恢复完整字典类型列表

### 修复要求

#### 3.1 初始状态

1. 字典管理页面初始加载时，不显示右侧字典项区域。
2. 初始状态只展示完整的字典类型列表。
3. 初始列表应保留完整字段，适合作为主列表浏览。

#### 3.2 打开字典项区域

1. 保留操作栏中的 `查看项` 按钮。
2. 点击 `查看项` 按钮时，打开右侧字典项区域。
3. 除了点击 `查看项` 按钮外，点击字典类型列表中的任意一行，也应打开右侧字典项区域。
4. 打开后应能明确知道当前选中的字典类型。

#### 3.3 打开后的左右布局

1. 打开右侧字典项区域后，左侧字典类型列表应压缩展示。
2. 左侧压缩列表只保留关键字段，例如：
   - 字典类型
   - 状态
   - 属性
3. 其他字段可以隐藏或省略，避免页面拥挤。
4. 右侧字典项区域应整体更靠左、更宽。
5. 右侧字典项区域应获得更多展示空间，方便查看字典项表格内容。

#### 3.4 关闭右侧字典项区域

1. 字典项区域需要提供明显的 `关闭` 按钮。
2. 点击关闭后：
   - 右侧字典项区域隐藏
   - 页面恢复为完整的字典类型列表
   - 左侧压缩状态取消
3. 关闭后不应残留布局错位或空白区域。

### 验收标准

- 初始进入字典管理页面，只显示完整字典类型列表。
- 点击 `查看项` 可以打开右侧字典项区域。
- 点击字典类型列表任意一行也可以打开右侧字典项区域。
- 打开字典项后，左侧字典类型列表压缩展示。
- 右侧字典项区域宽度更合理，内容展示空间更充足。
- 点击关闭按钮后，恢复完整字典类型列表。
- 执行 `npm run build` 通过。

## 四、创建时间与日志时间格式优化

### 问题描述

当前多个页面的时间展示不够符合实际使用需求。

需要区分处理：

1. 普通管理页面中的 `创建时间` 只显示日期即可。
2. 日志页面需要显示完整日期时间，并使用中国常用格式。
3. 日志页面时间列宽度需要足够，避免换行。

### 修复范围

普通管理页面：

- 字典管理页面
- 用户管理页面
- 角色管理页面

日志页面：

- 两个日志页面，按当前项目已有页面为准，例如登录日志、操作日志等。

### 修复要求

#### 4.1 普通管理页面创建时间

以下页面中的 `创建时间` 只显示日期：

- 字典管理页面
- 用户管理页面
- 角色管理页面

推荐格式：

```txt
YYYY-MM-DD
```

不需要显示：

```txt
HH:mm:ss
```

#### 4.2 日志页面时间格式

两个日志页面中的时间字段需要格式化为中国常用格式。

推荐格式：

```txt
YYYY-MM-DD HH:mm:ss
```

或者如果项目中已有统一时间格式工具，应优先复用现有工具。

#### 4.3 日志时间列宽度

1. 日志页面时间列宽度需要足够。
2. 日期和时间必须在一行完整显示。
3. 不允许出现日期时间换行导致表格观感变差。
4. 可以通过列宽、`white-space: nowrap` 或现有表格列配置解决。

### 验收标准

- 字典管理、用户管理、角色管理页面的 `创建时间` 只显示日期。
- 两个日志页面时间显示为完整日期时间。
- 日志页面时间符合中国常用阅读习惯。
- 日志页面时间列不换行。
- 执行 `npm run build` 通过。

## 五、用户管理重置密码报错修复

### 问题描述

用户管理页面中，点击某一行记录操作栏的 `重置` 按钮，确认重置密码后页面报错：

```txt
Unexpected Application Error!
Objects are not valid as a React child (found: object with keys {password}).
If you meant to render a collection of children, use an array instead.
```

错误关键点：

```txt
Objects are not valid as a React child (found: object with keys {password})
```

初步判断：

重置密码接口、mock 函数或业务函数返回了对象，例如：

```ts
{
  password: "123456"
}
```

但前端把整个对象直接作为 React 子节点渲染了，例如：

```tsx
message.success(result)
```

或：

```tsx
<div>{result}</div>
```

React 不能直接渲染对象，因此导致页面崩溃。

### 修复要求

1. 检查用户管理页面重置密码相关逻辑。
2. 检查相关 API、mock、service、页面组件中的返回值处理。
3. 如果重置密码返回对象，例如 `{ password: string }`，不能直接渲染整个对象。
4. 应显式读取 `password` 字段，或统一展示成功提示文案。
5. 所有 message、toast、Modal、JSX 内容中，都不能直接渲染对象。

### 推荐处理方式

如果需要显示新密码：

```tsx
message.success(`密码重置成功，新密码：${result.password}`)
```

如果不需要显示新密码：

```tsx
message.success("密码重置成功")
```

如果结果结构不稳定，应做兼容处理：

```tsx
const newPassword =
  typeof result === "object" && result !== null && "password" in result
    ? String(result.password)
    : ""

message.success(
  newPassword ? `密码重置成功，新密码：${newPassword}` : "密码重置成功"
)
```

### 额外优化建议

React Router 当前直接显示了开发错误页：

```txt
Unexpected Application Error!
Hey developer 👋
You can provide a way better UX than this when your app throws errors by providing your own ErrorBoundary or errorElement prop on your route.
```

建议补充：

1. 路由级 `errorElement`
2. 或应用级 `ErrorBoundary`

目的：

- 避免运行时错误直接显示开发错误页
- 给用户展示更友好的错误页面
- 提高后台系统稳定性体验

### 验收标准

- 点击用户管理页面任意用户的 `重置` 按钮。
- 确认重置密码后，页面不崩溃。
- 成功提示正常显示。
- 不再出现 `Objects are not valid as a React child`。
- 如果显示新密码，只显示字符串，不直接渲染对象。
- 如实现 ErrorBoundary 或 errorElement，运行时错误页面应更友好。
- 执行 `npm run build` 通过。

# AGENTS.md

## 角色

你是本项目的前端开发助手，负责协助开发一个基于 React 的后台管理系统前端模板。

## 交流要求

- 始终使用中文与用户交流。
- 解释、总结、计划和说明使用中文。
- 代码命名可使用英文。
- 不主动切换为英文。

## 项目定位

本项目是一个 React Admin 前端基础模板，用于后续复用到不同后台管理系统中。

第一目标是保持：

- 结构清晰
- 风格统一
- 易于扩展
- 易于维护
- 不过度设计

## 技术方向

默认采用以下技术方向：

- React
- TypeScript
- Vite
- Tailwind CSS
- shadcn/ui
- React Router
- Zustand

如项目实际依赖与以上不一致，以项目现有代码和用户最新要求为准。

## UI/UX 规范

React Admin UI/UX 主规范位于：

`design-system/MASTER.md`

开发任何页面、布局、组件和样式时，必须优先遵循该规范。

尤其需要遵守：

- 后台管理系统风格，不做营销官网风格。
- 保持中性、专业、高密度、弱装饰。
- 页面结构、间距、配色、圆角、阴影、字体层级应保持统一。
- 优先使用 design tokens，不在页面中随意写死颜色、间距、圆角、阴影。
- shadcn/ui 只作为基础组件来源，最终视觉以 `design-system/MASTER.md` 为准。

## 开发原则

- 先理解现有结构，再修改代码。
- 优先复用已有组件。
- 新增组件前，先判断是否能扩展已有组件。
- 保持组件职责单一。
- 避免过度封装。
- 避免一次性引入复杂功能。
- 不随意改变既有目录结构和公共 API。
- 不实现需求外的功能。

## 页面开发规则

开发新页面时，应先判断页面类型：

- Dashboard
- List
- Form
- Detail
- Settings
- Exception

然后选择对应的页面骨架和公共组件。

同类页面应保持一致的结构、间距和交互方式。

## shadcn/ui 使用规则

- 可以使用 shadcn/ui 作为基础组件。
- 不直接照搬默认视觉风格。
- 应基于项目设计规范进行样式适配。
- 页面级功能应优先封装为业务组件，而不是直接堆砌基础组件。

## 代码质量

- 使用 TypeScript。
- 保持命名清晰。
- 保持文件职责清晰。
- 删除无用代码。
- 避免重复逻辑。
- 避免引入不必要依赖。
- 修改完成后应说明修改内容和验证方式。

## 执行习惯

在执行较大改动前，先简要说明计划。

完成后总结：

- 修改了哪些文件
- 实现了什么
- 如何运行或验证
- 是否存在未完成事项
